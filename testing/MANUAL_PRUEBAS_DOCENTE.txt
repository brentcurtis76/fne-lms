0.1 ¿QUÉ ES ESTE MANUAL?
-------------------------
Este manual guía las pruebas del rol "Docente" en el sistema FNE LMS.
Un docente es un usuario que:
  - Accede a cursos y rutas de aprendizaje asignadas
  - Completa lecciones con diferentes tipos de contenido
  - Realiza tareas individuales y grupales
  - Colabora con otros docentes en comunidades de crecimiento
  - Gestiona su perfil y configuración personal

Las pruebas cubren 5 flujos principales:
  1. Autenticación (login, recuperación de contraseña, perfil)
  2. Cursos individuales (ver, navegar, completar lecciones)
  3. Rutas de aprendizaje (ver rutas asignadas, progreso)
  4. Tareas (individuales y grupales, envío, retroalimentación)
  5. Espacio colaborativo (mensajería, workspace, comunidades)

PARTE 4: PROBLEMAS COMUNES Y SOLUCIONES
================================================================================

Esta sección lista problemas típicos que pueden ocurrir durante las pruebas
y sus posibles soluciones.


PROBLEMA 1: No puedo iniciar sesión - "Credenciales inválidas"
---------------------------------------------------------------
POSIBLES CAUSAS:
  - Email o contraseña incorrectos
  - Usuario no existe en la base de datos
  - Usuario fue eliminado o desactivado

SOLUCIONES:
  1. Verificar que el email sea exacto: docente.test1@fne-lms.test
  2. Verificar que la contraseña sea: Prueba2025! (con mayúscula y signo)
  3. Consultar con el equipo técnico si los usuarios fueron creados
  4. Ejecutar el script 01-create-docente-test-users.sql nuevamente


PROBLEMA 2: El sidebar no muestra "Espacio Colaborativo"
---------------------------------------------------------
POSIBLES CAUSAS:
  - El usuario NO tiene comunidad asignada
  - El usuario logueado es test4 o test5 (sin comunidad)

SOLUCIONES:
  1. Verificar qué usuario está logueado (revisar nombre en sidebar)
  2. Si es test4 o test5, esto es ESPERADO (no tienen comunidad)
  3. Cambiar a usuario test1, test2 o test3 para ver la opción
  4. Consultar con técnico para asignar comunidad si es necesario


PROBLEMA 3: No veo cursos en "Mi Aprendizaje"
----------------------------------------------
POSIBLES CAUSAS:
  - No se ejecutó el script de asignación de cursos
  - El usuario no tiene cursos asignados
  - Error en la tabla `course_assignments` o `course_enrollments`

SOLUCIONES:
  1. Ejecutar el script 02-assign-courses-to-docentes.sql
  2. Verificar con el equipo técnico que hay cursos disponibles en el sistema
  3. Ejecutar query de verificación (03-verify-docente-setup.sql)
  4. Revisar logs de la consola del navegador (F12 → Console)


PROBLEMA 4: El progreso de lecciones no se guarda
--------------------------------------------------
POSIBLES CAUSAS:
  - Error en la API de progreso
  - Tabla `lesson_progress` tiene restricciones o problemas
  - Problema de conectividad con la base de datos

SOLUCIONES:
  1. Abrir consola del navegador (F12) y revisar errores
  2. Verificar que la petición POST se envía correctamente (pestaña Network)
  3. Consultar con técnico para revisar logs del servidor
  4. Intentar con otro usuario para descartar problema específico


PROBLEMA 5: Las tareas grupales NO aparecen
--------------------------------------------
POSIBLES CAUSAS:
  - El usuario NO está inscrito en el curso
  - El curso no tiene bloques de tipo "group-assignment"
  - El bloque de tarea grupal tiene `is_visible = false`

SOLUCIONES:
  1. Verificar que el usuario está inscrito en el curso (ver "Mi Aprendizaje")
  2. Consultar con técnico para verificar que el curso tiene tareas grupales
  3. Revisar con técnico la tabla `blocks` y filtrar por type='group-assignment'
  4. NOTA: La comunidad NO afecta visibilidad de tareas, solo afecta workspace


PROBLEMA 6: No puedo enviar mensajes en el workspace
-----------------------------------------------------
POSIBLES CAUSAS:
  - La funcionalidad de mensajería no está implementada
  - Error en la API de mensajes
  - Permisos insuficientes

SOLUCIONES:
  1. Consultar con técnico si la mensajería está implementada
  2. Si NO está implementada, marcar la prueba como OMITIDA
  3. Revisar consola del navegador (F12) para errores
  4. Verificar que el usuario tiene comunidad activa


PROBLEMA 7: Las notificaciones no aparecen
-------------------------------------------
POSIBLES CAUSAS:
  - No hay acciones recientes que generen notificaciones
  - El sistema de notificaciones usa Realtime y no está conectado
  - El usuario tiene preferencias de notificaciones desactivadas

SOLUCIONES:
  1. Realizar una acción que genere notificación (enviar tarea, completar lección)
  2. Esperar 10-15 segundos y refrescar el panel de notificaciones
  3. Revisar consola del navegador para errores de conexión
  4. Consultar con técnico para verificar suscripciones de Supabase Realtime


PROBLEMA 8: Error "No autorizado" al acceder a páginas
-------------------------------------------------------
POSIBLES CAUSAS:
  - La sesión expiró
  - Token de autenticación inválido
  - Problema con cookies del navegador

SOLUCIONES:
  1. Cerrar sesión y volver a iniciar sesión
  2. Limpiar cookies del navegador (Ctrl+Shift+Delete)
  3. Probar en modo incógnito/privado
  4. Consultar con técnico para revisar configuración de Supabase Auth
